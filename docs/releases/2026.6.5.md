# Therion Studio 2026.6.5

Draft release notes for tag `v2026.6.5`.

## Highlights

- Added Mapiah background transform support for TH2 map editing, including transform metadata parsing, editing, undo/redo, and export-oriented metadata updates.
- Improved raster background rendering so image layers stay sharp while zooming instead of being limited by the initial display resolution.
- Added background layer visibility regression coverage and expanded background metadata tests for mixed XTherion/Mapiah workflows.
- Refined sidebar internals by extracting project browser and search sidebar workflow code from `MainWindow`, keeping visible behavior stable while reducing UI shell complexity.
- Centralized shared UI style tokens and stylesheet builders for more consistent control sizing and segmented-control styling.
- Added the shared `TherionCoreQTests` QTest runner and migrated touched core source/text rule tests to QTest while keeping CTest as the test runner.
- Strengthened structure guardrails against map-editor source mutation bypasses.
- Reorganized planning documents under `plans/`, renamed the app specification to `SPECIFICATION.md`, and kept `WORKLOG.md` focused on active work.
- Centralized map/text undo ownership state transitions in the map undo arbitration service as a step toward durable transaction ownership metadata.

## Known Limitations

- French, German, Italian, Spanish, and Portuguese are staged for translation work but are not yet user-selectable.
- Old-project Therion/Metapost crash regression coverage still needs a reproducible fixture before it can be automated.
