# Therion Studio 2026.6.8

## Highlights

- Fixed loaded raster/XVI background layers in the map editor so viewport-only actions no longer refresh the background layer model, preventing Objects inspector blinking and avoiding interruption of line vertex/control-handle interaction.
- Preserved manually collapsed scraps in the map Objects inspector across refreshes and text navigation, while still expanding the containing scrap when a map object inside it is selected.
- Added a persisted `Auto-collapse/expand Scraps` toggle to the map Objects inspector for users who prefer the inspector to keep only the selected object's scrap expanded.
- Remember recent pending point, line, and area type/subtype choices as quick drawing defaults while the map editor stays open.
- Auto-advance pending `point station` names while drawing repeated station points.
- Keep pending line/area drawing options visible and allow setting the next line point smooth/subtype before each click, with slope-only orientation and left-size controls shown only for `line slope`.
- Treat line-point subtype as current-draft state only: repeated vertices no longer duplicate identical `subtype` rows, and each new line or area starts with an empty line-point subtype.
- Keep repeated point, line, and area drawing in the pending-object inspector after each insert so recent choices and draft fields stay available for the next object.
- Keep the committed line or area selected when `Esc` finishes drawing, and move the Raw editor cursor to that inserted object.
- Preserve line selection when editing the selected line type/subtype from the Selection inspector.
